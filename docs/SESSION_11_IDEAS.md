# Session 11 — backlog ideas (post-launch)

Park these until after launch validation (test purchase, DNS, invites, first real users).

## Express Pass / Lightning Lane / booked time slots

**Why it matters:** Many guests plan around Individual Lightning Lane, Virtual Queue windows, and dining or show reservations. Surfacing that in Smart Plan could reduce day-of surprises.

**Why not before launch:** Needs product decisions (who can edit, how it interacts with entitlements), schema or structured fields for “pinned” events, UI to toggle or edit slots, and prompt changes so the model does not hallucinate booking times. Expect several hours plus QA; every pre-launch feature competes with unshipped launch checklist items.

**When to revisit:** After you have signal from real trips (which parks, which pass products, how often users ask for LL vs dining vs rope drop).
